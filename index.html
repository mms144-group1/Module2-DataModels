Introduction to Data Models

Data model refers to the logical design and structure of a database. This means that the selected data model will determine the way data can be stored, accessed, and updated in a database management system (DBMS). Data models can also define the types of relationships a DBMS can present/process, along with the constraints that apply to those functions.

We will introduce popular data models and each model will be discussed in terms of how their design and structure works. Additionally, we will evaluate each model type based on their functional differences. This includes their corresponding advantages and disadvantages.

Through this, we hope to achieve the goal of clearly explaining the distinctions between the presented data models to help you make informed decisions on what model type to use for your DBMS.

Flat Files

Flat file database is a type of database that stores data in a single table of data files. Flat file databases are generally in plain-text form, where each line holds only one record.

Hierarchical Database

As the name suggests, the hierarchical database model is most appropriate for use cases in which the main focus of information gathering is based on a concrete hierarchy.

  • Learn more

Network Model

This model addresses major drawbacks of the hierarchical model in which a member can have multiple parents. In this model, the Parent entity is called “Owner”, and the Child entity is called “Member/s”.

  • Learn more

Relational Model

The relational model illustrates the relations of records and attributes through the rows and columns of a table.

  • Learn more

Object-oriented Model

An object-oriented database model follows the concepts of object-oriented programming in visualizing relationships between data which allows for wider possibilities in storing and manipulating a variety of data that is beyond the limits of relational model.

  • Learn more

Object-relational Model

Combination of object-oriented database model and a relational database model. As a result, it supports objects, classes, inheritance, and other features found in object-oriented model. As well as data types, tabular structures, and other features found in relational model.

Conclusion

These are just some of the widely used model types and there are no doubt other types out there. Each will have its design and purpose. As database management continues to evolve, so too will the development of these data models.
It is then our duty, as hopeful multimedia professionals, to continue to educate ourselves about them.

Flat files

This model type's simplicity allows ease of use and understanding, but also renders it unsuitable for complex data types and operations

Hierarchical model

The tree-like structure makes this ideal for 1:N relationships, but child directories being limited to one parent limits it from M:N and other complex relationships.

Network Model

Member directories having multiple owners enables many-to-many relationships, but ease of use and maintenance becomes more difficult as the number of sets increases

Relational model

Data integrity and consistency is achieved through unique tuples and normalization limits redundancy. This model, however, is unable to handle complex data types or operations

Object-oriented model

Storing data in objects enables complex operations that are unavailable with the relational model. This model, however, is too complex for simpler tasks

Object-relational model

This hybrid model can handle complex data types like BLOBs, but its complexity presents challenges in terms of handling and maintenance
